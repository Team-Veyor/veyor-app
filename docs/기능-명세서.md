## 1. 로그인

**동작**

- 카카오 로그인 SDK 실행
- 로그인 성공 시 회원 정보 조회

**분기**

- 신규 회원 : 온보딩 이동
- 기존 회원 : 홈 이동

**예외**

- 로그인 취소
    - → 로그인 화면 유지
- 로그인 실패
    - → "로그인에 실패했습니다. 다시 시도해주세요." 토스트 표시
- 네트워크 오류
    - → "네트워크 연결을 확인해주세요." 토스트 표시

- Top Navigation
    - `Back` 버튼

### 4-2. 계좌 정보 관리

#### 상태 A : 등록 계좌 없음

**노출**

- `내 계좌 추가하기` 버튼

**기능**

- `내 계좌 추가하기` 버튼 클릭시 A. 계좌 정보 입력 페이지로 이동

#### 상태 B : 등록 계좌 있음

**노출**

- 등록된 계좌 카드
    - 대표 계좌(1개)
        - 은행 이름
        - 대표 계좌 badge
        - 계좌 번호(마스킹)
        - `더보기` 버튼
    - 이외
        - 은행 이름
        - 계좌 번호(마스킹)
        - `더보기` 버튼
- `내 계좌 추가하기` 버튼

**기능**

- 등록된 계좌 카드 - `더보기` 버튼 클릭시 메뉴 노출
    - 대표 계좌(1개)
        - 수정
            - `수정` 버튼 클릭 → A. 계좌 정보 입력 페이지로 이동(입력된 상태)
        - 삭제
            - `삭제` 버튼 클릭 → "등록된 계좌를 삭제할까요?" 다이얼로그 표시
    - 이외
        - `대표 계좌로 선택`
            - 버튼 클릭 → "대표 계좌로 선택할까요?" 다이얼로그 표시
        - `수정`
            - 버튼 클릭 → A. 계좌 정보 입력 페이지로 이동(입력된 상태)
        - `삭제`
            - 버튼 클릭 → "등록된 계좌를 삭제할까요?" 다이얼로그 표시

**정책**

- 대표 계좌는 1개만 설정 가능
- 대표 계좌 삭제시
    - 등록된 다른 계좌가 있는 경우
        - 1개: 대표 계좌로 자동 지정
        - 2개 이상: 대표 계좌 선택 Bottom Sheet 표시
    - 등록된 계좌가 없는 경우
        - A. 계좌 정보 입력 페이지로 이동

## A. 계좌 정보 입력

**노출**

- 콜아웃
    - 안내 사항
- **입력 항목**
    - 은행
    - 계좌번호
    - 예금주명
- `저장` 버튼

**기능**

- **입력 항목**
    - 은행
        - 클릭시 은행 선택 Bottom Sheet 표시
    - 계좌번호
    - 예금주명
- **`저장` 버튼**
    - 필수값 입력 시 활성화
    - 클릭 시 → 계좌 정보 저장, 홈 화면(상태 B : 등록 계좌 있음)으로 이동
- **클립보드 감지**
    - 조건
        - 클립보드에 계좌번호 형식 데이터 존재
    - 동작
        - "계좌번호를 자동으로 입력할까요?" 다이얼로그 노출
            - `예` 선택 → 계좌번호 자동 입력
            - `아니요` 선택 → 다이얼로그 닫기

**예외**

- 은행/계좌번호/예금주명 미입력 시
    - → `저장` 버튼 비활성화
- 저장 실패
    - → "저장에 실패했습니다. 다시 시도해주세요." 토스트 표시

## B. 설문 완료

### B-1. 인증

> 외부 설문 제출 후 참여 완료를 처리하는 페이지

**진입 경로**

- 홈 화면(상태 B : 등록 계좌 있음) → `시작하기` 버튼 클릭 → 외부 설문 이동 → 외부 설문 제출 → 설문 마지막 페이지 내 서비스 복귀 링크 클릭 → B-1. 인증 페이지 진입

**노출**

- 설문 제목
- 예상 소요 시간
- 리워드 금액
- `설문 완료` 버튼

**기능**

- `설문 완료` 버튼 클릭 시
    - 동작
        - B-2. 완료 페이지로 이동
        - 설문 참여 완료 상태 저장
        - 참여 이력 생성
        - 리워드 지급 대기 상태 생성
- `Back` 버튼 클릭 시 → 홈 화면으로 이동

### B-2. 완료

**진입 경로**

- B-1. 인증 페이지 → `설문 완료` 버튼 클릭 → B-2. 완료 페이지 진입

**노출**

- `홈 화면 이동` 버튼

**기능**

- `홈 화면 이동` 버튼 클릭 시 → 홈 화면으로 이동
- `Back` 버튼 클릭 시 → 홈 화면으로 이동

### 예외

- 이미 참여 완료한 설문
    - → "이미 참여한 설문입니다." 토스트 표시
- 만료된 설문
    - → "참여 기간이 지난 설문입니다." 토스트 표시
- 서버 오류
    - → "인증할 수 없습니다. 다시 시도해주세요." 토스트 표시

## 주요 플로우

1. **계좌 등록**
    - 홈 (계좌 미등록) → 등록하기 → A. 계좌 등록 → 저장 → 홈 (설문 참여 가능)

2. **설문 참여**
    - 홈 → 시작하기 → 외부 설문 (외부 설문 제출 → 서비스 복귀 링크 클릭) → B. 설문 완료 (설문 완료 버튼 클릭) → 참여 완료 처리 → 홈 이동

3. **참여 내역 조회**
    - 홈 → 참여 내역 보기 → 4-1 참여 내역 페이지 이동
    - 내 정보 → 참여 내역 보기 → 4-1 참여 내역 페이지 이동

## 운영 정책

**리워드 지급 정책**

- 설문 완료 버튼을 클릭한 사용자만 지급 대상에 포함
- 리워드는 참여 완료 후 당일 00:30 ~ 다음날 오전 중 지급

**연속 참여 정책**

- 하루 1회 참여 가능
- 하루라도 참여하지 않으면 연속 참여 일수는 0일로 초기화
- 설문 완료 처리된 경우에만 연속 참여로 인정

**계좌 정책**

- 계좌 정보는 리워드 지급 목적으로만 사용
- 예금주명은 계좌 실명과 동일해야 함
- 계좌 정보 오입력 시 지급이 지연될 수 있음

**설문 완료 정책**

- 외부 설문 제출만으로는 참여 완료 처리되지 않음
- 서비스 복귀 후 설문 완료 버튼 클릭까지 완료해야 참여 인정
- ~~운영자 검수 결과 응답이 확인되지 않을 경우 리워드 지급 대상에서 제외될 수 있음~~

**탈퇴 정책**

- 탈퇴 시 계정 정보 삭제
- 활동 내역 삭제
- 삭제된 정보는 복구 불가

# 2차 MVP 기획

## 2차 MVP 개발 목적

1. 운영 효율화
2. 개선을 위한 추적 데이터를 얻기 위함
3. 오픈 채팅방 홍보를 통한 확장의 한계

## 개발 형태

- PWA(Progressive Web App)
    - 이유
        - 홈 화면 설치 가능
        - 앱스토어 심사 시간 절약
        - 앱 개발 비용 절약

| 구분 | 일반 웹 | PWA | 네이티브 앱 |
| --- | --- | --- | --- |
| 설치 | X | O | O |
| 앱스토어 심사 | X | X | O |
| 푸시 알림 | 제한적 | 가능 | 가능 |
| 오프라인 | 거의 X | 일부 가능 | 가능 |
| 성능 | 보통 | 중간~좋음 | 가장 좋음 |
| 개발 비용 | 낮음 | 낮음~중간 | 높음 |

## AS-IS

**운영자 관점**

- 설문 요청자 관리
    - **설문 게시 전**
        - 설문 검토(마감일, 대상, 비용)
        - 설문 게시 가능 여부 및 비용 안내
        - 진행 전 안내
    - **설문 게시 후**
        - 참여자 인증 캡처
        - 비용 계산
        - 설문 결과 및 입금 안내
- 참여자 관리
    - **설문 게시 전**
        - 당일 설문 정보 작성
        - 오픈채팅봇 멘트 수정
    - **설문 게시 후**
        - 리워드 지급(계좌/카카오페이)
        - 활동 로그 추적 및 통계

**참여자 관점**

- **설문 참여 후**
    - 인증
    - 카카오페이) 페이 머니 받기

**요청자 관점**

- 설문 게시 전
    - 진행 여부 회신
- 설문 게시 후
    - 이용료 입금

>     - '1일 1설문으로 하면 매일 들어온다'를 이미 MVP에서 검증했기 때문에 유지.

- 로그인 `new`
    - 카카오 소셜 로그인
    - 자동 로그인
- 설문 게시

    **1안) 우리 서비스 내에서 변환해서 게시**

    - 설문 참여자 - 완료 버튼을 통해 참여 인증
    - 설문 요청자 - 기존 링크와 설문 결과 취합 불가

    **2안) 기존 링크 그대로 게시**

    - 설문 참여자 - 완료 버튼을 통해 참여 인증
    - 설문 요청자 - 기존 링크와 설문 결과 취합 가능

    **3안) iframe**

    - Iframe(아이프레임, Inline Frame)은 **현재 웹 페이지 안에 다른 외부 웹 페이지나 콘텐츠(예: 유튜브 영상, 구글 지도)를 독립적으로 삽입하여 표시할 수 있게 해주는 HTML 요소**입니다.

    > 💡 **🚀 제안 : 아래 인증 방식 중 하나라도 가능하면, 2안으로 진행**
    >
    > - 이유
    >     - 운영 효율화(설문 변환 리소스 절약)
    >     - 작고 빠르게 실험(최소 기능만 개발 → 개발 리소스 절약)
    >     - 설문 요청 진입 장벽 낮음(기존 링크와 결과 취합 가능)
    > - 인증 방식
    >     1. '**완료 링크**'를 통한 인증
    >     2. '**완료 코드**'를 통한 인증

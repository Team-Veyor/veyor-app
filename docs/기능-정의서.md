> 화면 단위 상세 동작/예외는 `기능-명세서.md`를, 기획 의도·정책 배경은 `2차-MVP-기획.md`를 함께 참고하세요.

## 1. 서비스 한눈에 보기

```
[비회원] ──카카오 로그인──▶ [신규] 온보딩(정보입력→약관동의→서비스소개) ──▶ 홈
                          [기존] ─────────────────────────────────────▶ 홈

[홈] 계좌 미등록 ──등록하기──▶ 계좌 등록 ──▶ 홈(설문 참여 가능)
[홈] 설문 참여 ──시작하기──▶ 외부 설문 ──복귀 링크──▶ 설문 완료 인증 ──▶ 리워드 지급 대기
[홈] ──매일 1설문, 연속 참여 현황──▶ 참여 내역 / 내 정보 / 계좌 관리
```

### 사용자 역할

| 역할 | 본 서비스(앱) 범위 | 비고 |
| --- | --- | --- |
| **참여자(User)** | ✅ 앱의 주 사용자 | 설문 참여 → 리워드 수령 |
| 운영자(Admin) | ⛔ 별도(수기/배치 운영) | 설문 등록, CSV 정산, 알림톡 발송 |
| 요청자(Client) | ⛔ 앱 범위 외 | 설문 의뢰·이용료 입금 |

> 2차 MVP의 앱 개발 범위는 **참여자 경험**에 집중. 운영자 작업은 당분간 수기/배치(예: 매일 CSV 추출 후 은행 일괄 정산).

## 2. 기능 목록 (도메인별)

기능 ID 체계: `도메인-순번` (예: `AUTH-1`). 프론트/백엔드 이슈·PR에서 참조용으로 사용 가능.

### 2.1 인증 (AUTH)

| ID | 기능 | 설명 |
| --- | --- | --- |
| AUTH-1 | 카카오 소셜 로그인 | 카카오 OAuth로 로그인. Supabase Auth가 인증 담당, 서버는 JWT 검증만 |
| AUTH-2 | 신규/기존 회원 분기 | 로그인 성공 시 회원 정보 조회 → 신규: 온보딩 / 기존: 홈 |
| AUTH-3 | 자동 로그인 | 세션 유지(재방문 시 재로그인 불필요) |
| AUTH-4 | 로그아웃 | 세션 종료 → 로그인 화면 이동 |
| AUTH-5 | 회원 탈퇴 | 계정/활동 내역 삭제(복구 불가) → 로그인 화면 이동 |

**예외 처리**
- 로그인 취소 → 로그인 화면 유지
- 로그인 실패 → "로그인에 실패했습니다. 다시 시도해주세요." 토스트
- 네트워크 오류 → "네트워크 연결을 확인해주세요." 토스트

---

### 2.2 온보딩 (ONBOARD)

| ID | 기능 | 설명 |
| --- | --- | --- |
| ONBOARD-1 | 기본 정보 입력 | 출생연도, 성별 입력. 필수값 입력 시 `다음` 활성화 |
| ONBOARD-2 | 약관 동의 | 필수(개인정보 수집·이용, 서비스 이용약관) + 선택(마케팅 수신). 전체/개별 동의, 필수 모두 동의 시 `확인` 활성화 |
| ONBOARD-3 | 서비스 소개 | 3페이지(① 매일 1~5분 설문 ② 익일 계좌 입금 ③ 사용자 후기). 좌우 스와이프 + 시간차 자동 전환, `시작하기` → 홈 |

**저장 데이터**: 출생연도, 성별, 약관별 동의 여부(+동의 시각)

> 입력한 연령/성별은 **설문 개인화**(연령·성별 타깃 설문 노출)에 사용. (기획 TO-BE)

---

### 2.3 홈 (HOME)

홈은 사용자 상태에 따라 3가지로 분기.

| 상태 | 노출 | 핵심 동작 |
| --- | --- | --- |
| **A. 계좌 미등록** | 계좌 등록 카드 + `등록하기` | `등록하기` → 계좌 정보 입력 |
| **B. 설문 참여 가능** | 오늘 설문 카드(제목·소요시간·리워드·`시작하기`) + 연속 참여 현황 카드 | `시작하기` → 외부 설문 / `참여 내역 보기` → 참여 내역 |
| **C. 설문 참여 완료** | B와 동일, 진행 상황 "완료" 표시 | 동일 |

| ID | 기능 | 설명 |
| --- | --- | --- |
| HOME-1 | 오늘 설문 카드 | 당일 설문 1건 노출(진행 전/완료), 시작하기 → 외부 설문 링크 |
| HOME-2 | 연속 참여 현황 | 연속 일수, 일수별 멘트, 누적 금액, 주간 현황 그래픽 |
| HOME-3 | 계좌 미등록 유도 | 계좌 없으면 등록 카드 우선 노출 |

---

### 2.4 설문 참여 / 완료 (SURVEY)

| ID | 기능 | 설명 |
| --- | --- | --- |
| SURVEY-1 | 외부 설문 연결 | 홈 `시작하기` → 기존 외부 설문 링크로 이동(2차 MVP: 기존 링크 그대로 게시 = 2안) |
| SURVEY-2 | 복귀 인증 | 외부 설문 제출 후 복귀 링크/코드로 서비스 복귀 → 인증 페이지 진입 |
| SURVEY-3 | 설문 완료 처리 | `설문 완료` 클릭 시: 참여 완료 상태 저장 + 참여 이력 생성 + **리워드 지급 대기** 생성 |
| SURVEY-4 | 완료 화면 | 완료 안내 → `홈 화면 이동` |

**핵심 정책**
- **외부 설문 제출만으로는 완료 아님.** 서비스 복귀 후 `설문 완료` 버튼 클릭까지 해야 참여 인정.
- 하루 1회만 참여 가능.

**예외 처리**
- 이미 참여한 설문 → "이미 참여한 설문입니다."
- 만료된 설문 → "참여 기간이 지난 설문입니다."
- 서버 오류 → "인증할 수 없습니다. 다시 시도해주세요."

---

### 2.5 리워드 / 참여 내역 (REWARD)

| ID | 기능 | 설명 |
| --- | --- | --- |
| REWARD-1 | 리워드 지급 대기 생성 | 설문 완료 시 지급 대기 레코드 생성 |
| REWARD-2 | 리워드 지급(정산) | 운영: 매일 CSV 추출 후 은행 일괄 업로드. 지급 시점: 완료 당일 00:30 ~ 익일 오전 |
| REWARD-3 | 누적 현황 | "내가 모은 돈"(누적 참여 횟수/금액) |
| REWARD-4 | 참여 내역 조회 | 캘린더(월 이동) + 참여 리스트(설문 제목·날짜·지급 금액) |
| REWARD-5 | 연속 참여 산정 | 완료 처리된 경우만 연속 인정, 하루라도 미참여 시 연속 0 초기화 |

---

### 2.6 계좌 관리 (ACCOUNT)

| ID | 기능 | 설명 |
| --- | --- | --- |
| ACCOUNT-1 | 계좌 등록 | 은행(선택 Bottom Sheet)·계좌번호·예금주명 입력 → 저장. 필수값 입력 시 `저장` 활성화 |
| ACCOUNT-2 | 클립보드 자동 입력 | 클립보드에 계좌번호 형식 존재 시 "자동 입력할까요?" 다이얼로그 |
| ACCOUNT-3 | 계좌 목록 조회 | 대표 계좌(badge) + 그 외 계좌, 계좌번호 마스킹 표시 |
| ACCOUNT-4 | 계좌 수정 | `수정` → 입력값 채워진 상태로 계좌 입력 페이지 |
| ACCOUNT-5 | 계좌 삭제 | 확인 다이얼로그 후 삭제(복구 불가) |
| ACCOUNT-6 | 대표 계좌 지정 | 대표 계좌는 **1개만**. 선택 시 확인 다이얼로그 |

**대표 계좌 삭제 정책**
- 다른 계좌 1개 → 자동 대표 지정
- 다른 계좌 2개 이상 → 대표 선택 Bottom Sheet
- 다른 계좌 없음 → 계좌 입력 페이지로 이동

**계좌 정책**: 리워드 지급 목적으로만 사용 / 예금주명은 실명과 동일 / 오입력 시 지급 지연 가능

**예외**: 미입력 시 저장 비활성화, 저장 실패 → "저장에 실패했습니다. 다시 시도해주세요."

### 2.7 내 정보 / 약관 (PROFILE)

| ID | 기능 | 설명 |
| --- | --- | --- |

| PROFILE-4 | 이용 약관 열람 | 서비스 이용약관 / 개인정보 수집·이용 / 개인정보처리방침 (아코디언) |

| PROFILE-6 | 로그아웃 / 탈퇴 | 다이얼로그 확인 후 처리 |

## 4. 데이터 모델 (개략)

> 확정 스키마가 아닌 **도메인 엔티티 초안**. 실제 컬럼/제약은 마이그레이션 설계 시 확정.

| 엔티티 | 핵심 필드(초안) | 비고 |
| --- | --- | --- |
| `users` | id, kakao_id, name, email, birth_year, gender, created_at, deleted_at | Supabase Auth 연동 |
| `consents` | user_id, type(privacy/terms/marketing), agreed, agreed_at | 약관 동의 이력 |
| `accounts` | id, user_id, bank, account_no(암호화/마스킹), holder_name, is_primary | 대표 1개 제약 |
| `surveys` | id, title, external_url, reward_amount, est_minutes, opens_at, expires_at, target(연령/성별) | 운영자 등록 |
| `participations` | id, user_id, survey_id, status, completed_at | 하루 1회·중복 방지 |
| `rewards` | id, participation_id, user_id, amount, status(pending/paid), paid_at | CSV 정산 연계 |
| `streaks` | user_id, count, last_participated_on | 연속 참여 산정 |

**주의 사항(보안/개인정보)**
- 계좌번호 등 민감정보는 **암호화 저장**, 표시는 **마스킹**.
- 탈퇴 시 계정/활동 내역 삭제(복구 불가) — 법적 보관 의무 데이터는 정책 확인 필요.

## 5. 외부 연동 / 운영

| 항목 | 내용 |
| --- | --- |
| 카카오 OAuth | 로그인 (Supabase Auth 경유) |
| 카카오 알림톡 | 설문 오픈/지급 완료/리마인드 |
| 외부 설문 도구 | 기존 설문 링크 그대로 게시(2안) + 완료 링크/코드 인증 |
| 리워드 정산 | 매일 CSV 추출 → 은행 일괄 업로드(수기/배치) |
| 오픈 채팅방 | 초기 유지(운영 소통·이탈 방어), 채팅 상담 바로가기 연결 |

## 6. 핵심 정책 요약

| 영역 | 정책 |
| --- | --- |
| 설문 | 1일 1설문 / 하루 1회 참여 / 완료 버튼 클릭까지 해야 인정 |
| 리워드 | 완료자만 지급 / 당일 00:30~익일 오전 지급 |
| 연속 참여 | 완료만 인정 / 미참여 시 0 초기화 |
| 계좌 | 지급 목적 한정 / 실명 일치 / 대표 1개 |
| 탈퇴 | 계정·활동 내역 삭제, 복구 불가 |

## 7. 기술 스택

| 구분 | 결정 |
| --- | --- |
| 프론트엔드 | Next.js 16 (PWA) — `apps/client` |
